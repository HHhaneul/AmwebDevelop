트랜잭션 제어와 세션

1. 트랜잭션이란?
- 작업 수행을 위해 나눌 수 없는 최소 수행 단위

- COMMIT 할 때까지가 하나의 트랜잭션
	예)
           	주문 -> 3개 상품
           	3개 상품 정보 -> 쿼리 3번 추가
           	주문서 정보 -> 쿼리 1번


2. 트랜잭션을 제어하는 명령어

3. COMMIT ☆★시험문제★☆

세션과 읽기 일관성의 의미
1. 세션이란?
	데이터베이스 연결 접속
	- 세션간 독립성 유지


2. 읽기 일관성


3. LOCK ☆★시험문제★☆
	- 이전 세션에서 동일한 레코드를 수정, 삭제
	- 다른 세션 LOCK
	- 이전 세션에서 COMMIT, ROLLBACK을 통해서 처리 완료 LOCK 해제
